INTRODUCTION                                              *dap-inline-values*

DAP Inline Values is a lightweight Neovim plugin that displays variable values
inline during debugging sessions. Position your cursor on any expression and
press a keybind to reveal its current value in the debugger context.

The plugin uses tree-sitter to intelligently detect expressions and works with
any DAP adapter (Python, Lua, JavaScript, etc.).

1. SETUP                                                  *dap-inline-values-setup*

Load the plugin with lazy.nvim:

>lua
    {
      "monster0506/dap-inline-values.nvim",
      dependencies = {
        "mfussenegger/nvim-dap",
        "nvim-treesitter/nvim-treesitter",
      },
      config = function()
        require("dap_inline_values").setup({
          filetypes = { "python", "lua", "javascript" },
        })
      end,
    }
<

==============================================================================
2. CONFIGURATION                                          *dap-inline-values-config*

Options passed to `setup()` override the plugin's defaults.
Default values are listed in |dap-inline-values-defaults|.

 filetypes
  List of filetypes where inline evaluation is enabled.
  Example: `{ "python", "lua", "javascript" }`

 keymaps
  Table of keybindings. Currently supports:
  - `evaluate`: Key to toggle inline value display (default: "M")

 value_prefix
  String prepended to displayed values (default: "-> ")
  Example: Set to "â†’ " for a different symbol.

==============================================================================
3. KEYBINDINGS                                            *dap-inline-values-keymaps*

Default key mappings (when in an enabled filetype during a debug session):

    M    Evaluate expression at cursor and display value inline
         Press again to hide the inline display

You can override the keymap using:

>lua
    keymaps = {
      evaluate = "<leader>de",
    }
<

==============================================================================
4. USAGE AND BEHAVIOR                                    *dap-inline-values-usage*

Prerequisites:
  1. A DAP session must be active (`:DapContinue`)
  2. Execution must be paused at a breakpoint
  3. The current buffer's filetype must be in your configured filetypes

To use:
  1. Position your cursor on a variable or expression
  2. Press the evaluate keybind (default: M)
  3. The current value appears inline next to the expression
  4. Press the keybind again to remove the inline display

Supported expressions:
  - Simple identifiers: `x`, `name`, `result`
  - Attributes: `person.age`, `circle.radius`
  - Subscripts: `numbers[0]`, `data["key"]`
  - Function calls: `len(text)`, `str(value)`

If a variable is not yet defined in the current scope, the plugin displays
an error message.

Values longer than 100 characters are automatically truncated with "...".

Below are the plugin defaults:

        evaluate = "M",

==============================================================================
REQUIREMENTS

- Neovim 0.9 or later
- nvim-dap (Debug Adapter Protocol)
- nvim-treesitter with parsers for your language

==============================================================================
NOTES

- Inline values are buffer-local and cleared when you switch buffers or stop
  debugging.
- The plugin uses tree-sitter to intelligently extract expressions, so
  coverage depends on your language's parser.
- All evaluations happen in the context of the currently paused stack frame.
